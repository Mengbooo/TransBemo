## 1. 产品简介

百度通用翻译API是基于先进自然语言处理技术打造的在线文本翻译服务，支持200+种语言互译及100+语种自动检测。用户只需提交待翻译文本，指定源语言（支持自动检测）和目标语言，即可获取翻译结果。该API适用于移动端、PC网站、智能硬件等多种场景，提供高效优质的多语言实时翻译服务。

## 2. 接入步骤

1. 使用您的百度账号登录百度翻译开放平台
2. 完成注册，并在开发者信息页面获得 APPID 和 密钥
3. 进行开发者认证（如仅需标准版可跳过）
4. 开通通用翻译API服务：[开通链接]
5. 参考技术文档和 Demo 编写代码

## 3. 接入技术指南

通用翻译API通过 HTTP 接口对外提供多语种互译服务。您只需要通过调用通用翻译API，传入待翻译的内容，并指定要翻译的源语言（支持源语言语种自动检测）和目标语言种类，就可以得到相应的翻译结果。

### API 地址
`https://fanyi-api.baidu.com/api/trans/vip/translate`

### 输入参数

| 字段名 | 类型 | 是否必填 | 描述 | 备注 |
|--------|------|----------|------|------|
| q | string | 是 | 请求翻译query | UTF-8编码，完成个人/企业认证后上限为6000字符 |
| from | string | 是 | 翻译源语言 | 可设置为auto（自动检测语言） |
| to | string | 是 | 翻译目标语言 | 不可设置为auto |
| appid | string | 是 | APPID | 可在开发者信息查看 |
| salt | string | 是 | 随机数 | 可为字母或数字的字符串 |
| sign | string | 是 | 签名 | appid+q+salt+密钥的MD5值 |
| needIntervene | integer | 否 | 判断是否需要使用自定义术语干预API | 1-是，0-否 |

### 鉴权签名（sign）生成方法

1. 将请求参数中的 APPID(appid)，待翻译文本(q)，随机数(salt)，以及平台分配的密钥按照 `appid+q+salt+密钥` 的顺序拼接得到字符串
2. 对字符串做 MD5，得到 32 位小写的 sign

## 6. 附加功能

### 6.1 自定义术语库

自定义术语库是百度翻译推出的 API 增值服务，如果您认为通用翻译API对于某些术语翻译不准确，可在"管理控制台-我的术语库"页面填写您认为正确的原文和译文。

**使用方法**：
- 在接口 URL 增加 `&needIntervene=1`
- 提交后需等待 10 分钟方可生效

**注意事项**：
- 术语干预并非简单的一一替换
- 仅适用于专有名词、术语的自定义
- 暂不支持针对 HTML 代码原样输出

### 6.2 词典、语音合成资源

百度翻译提供文本翻译-词典版服务，在通用翻译API基础上，翻译结果附带百万量级中英词典资源和语音合成资源。

## 7. 常见问题

### 7.1 如何在一次请求中翻译多个单词或多段文本？

使用换行符（`\n`）分隔要翻译的多个单词或文本。注意在发送请求前需对 `q` 字段做 URL encode。

### 7.2 什么是URL encode？

网络标准 RFC 1738 规定 URL 中只能使用英文字母、阿拉伯数字和某些标点符号。对于不在规定范围内的字符（如中文），需要通过 URL encode 编码。

### 7.3 字符数量如何统计？

以翻译的源语言字符长度为标准：
- 一个汉字、一个英文字母、一个标点符号均记为一个字符
- 空格、HTML 标签等均计入字符数

### 7.4 单次翻译请求的字符数限制？

- 单次翻译文本长度限定为 6000 字节以内（约 2000 个汉字）
- 高峰期长文本可能导致翻译超时
- 建议将长文本分多次请求
